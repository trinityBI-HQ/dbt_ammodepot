## Summary

Built and deployed a natural language chatbot that translates user questions into SQL against 6 Gold layer models via Snowflake Cortex Analyst. Users can ask questions like "What is total revenue today?" or "Which vendors have the longest lead times?" and get instant results.

**Deployed to:** `AD_ANALYTICS.OPS.ANALYST` (SiS container runtime, `sales_dashboard_pool`)

## Lessons Learned

### Process

| Lesson | Detail |
|--------|--------|
| **SDD workflow works for new technology** | Brainstorm → Define → Design → Build caught scope decisions early (Option A vs B vs C, 6 vs 13 models, separate app vs page) |
| **Automated testing is essential for LLM features** | Manual testing is too slow for 25+ questions; the `test_golden_questions.py` script saved hours of Snowsight clicking |
| **Iterate on YAML in small batches** | Each semantic view change requires DROP + recreate. Batching 5 fixes per iteration was more efficient than 1 fix at a time |

### Technical — Cortex Analyst Semantic Views

| Lesson | Detail |
|--------|--------|
| **`CREATE SEMANTIC VIEW ... AS $$` is invalid** | Use `SYSTEM$CREATE_SEMANTIC_VIEW_FROM_YAML('schema', $$yaml$$)` |
| **`ALTER SEMANTIC VIEW SET VERIFIED_QUERIES` is invalid** | Verified queries must be in the YAML body, not added separately |
| **`GRANT USAGE ON SEMANTIC VIEW` is invalid** | Use `GRANT SELECT ON VIEW` |
| **Dimension/fact `name` must match physical column name** | Cortex CTE aliases `physical AS logical`. If they differ, aggregations reference the physical name which was aliased away. Make `name` = physical (case-insensitive) to create no-op aliases |
| **Mixed-case quoted columns break Cortex** | D_PRODUCT's `"Caliber"`, `"Product Name"` etc. are incompatible. Solution: thin wrapper view (`int_product_analyst`) with UPPERCASE unquoted names |
| **`primary_key` and `relationship_columns` use logical dimension names** | Not physical column names. `right_column: product_id` (the dimension name), not `right_column: "Product ID"` |
| **Verified queries are the #1 accuracy lever** | Without them, Cortex generates its own SQL that may be semantically wrong (Q15: GP without freight cost subtraction, Q4: exact caliber match vs ILIKE) |
| **Caliber filtering needs ILIKE** | Caliber values include variants ("9mm", "9mm Luger", "9mm Luger +P"). Exact match `= '9mm'` misses most inventory. Add ILIKE guidance in the column description |
| **Container runtime: `SNOWFLAKE_ACCOUNT` env var lacks region** | Set to `iwb48385` instead of `iwb48385.us-east-1`. Must check `SNOWFLAKE_HOST` first or hardcode the full locator |
| **Container runtime: no EAI needed for Cortex API** | Internal Snowflake API calls work without External Access Integration from container runtime |

### Communication

| Lesson | Detail |
|--------|--------|
| **Client constraint shaped the entire architecture** | "No external LLM API keys" eliminated 2 of 3 options immediately — ask this question first |
| **Validate before claiming** | The first 15/15 pass was on execution success, not metric accuracy. Cross-validation caught Q15 at 26% off |

## Known Limitations (Phase 1)

| Limitation | Impact | Resolution |
|-----------|--------|------------|
| `DISCONTINUED` column returns 0 for filter queries | D_PRODUCT has NULLs, not FALSE | Phase 2: add `is_discontinued` boolean to int_product_analyst |
| Caliber exact match on some queries | Cortex sometimes uses `=` instead of `ILIKE` | Mitigated by description text + verified queries |
| No cross-model calculated metrics | Can't answer "stock-out risk in 30 days" | Phase 2 scope |
| No chart generation | Returns tables only | Phase 3 scope |
| 6 of 13 Gold models covered | 7 models deferred (cohort, shipment, etc.) | Phase 2+ expansion |
