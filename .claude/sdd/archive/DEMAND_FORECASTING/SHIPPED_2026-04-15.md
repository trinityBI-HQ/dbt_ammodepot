## Summary

Built a demand forecasting pipeline using Snowflake's native `SNOWFLAKE.ML.FORECAST` that predicts units sold by caliber for the next 30 days. Combined with current inventory and vendor lead times to generate stock-out risk alerts. Revenue forecast included as a bonus.

**Deployed to:** Streamlit Sales Dashboard Page 4 (`streamlit_app/pages/4_Forecast.py`) + weekly Snowflake Task

## Lessons Learned

### Technical — SNOWFLAKE.ML.FORECAST

| Lesson | Detail |
|--------|--------|
| **XSMALL is too slow for 115 series** | Training took 31 min on XSMALL. Changed to weekly schedule. For daily, would need SMALL warehouse (~$5/mo) |
| **Sparse series fail the entire model** | If ANY caliber has < 2 timestamps, `CREATE FORECAST` fails entirely — not per-series. Filter with `HAVING COUNT(DISTINCT date) >= 90` |
| **FORECASTING_PERIODS varies by series frequency** | FORECAST auto-detects each series' frequency. Low-volume calibers detected as weekly → 30 periods = 30 weeks, not 30 days. Solution: filter chart to `DATEADD(30, CURRENT_DATE())` |
| **INSERT OVERWRITE is atomic** | Only commits on success; if FORECAST fails, previous predictions remain. Good pattern for ML pipelines |
| **SYSTEM$REFERENCE fails in Python connector** | Can't call stored procedures that use SYSTEM$REFERENCE from snowflake-connector-python. Must run in Snowsight or Snowflake Task |
| **Plotly `title=None` renders as "undefined" in SiS** | Use `title=""` (empty string) instead. Move titles to `st.subheader()` above the chart |

### Process

| Lesson | Detail |
|--------|--------|
| **dbt should own all views** | Started with manual DDL, immediately moved to dbt models (int_daily_sales_by_caliber). Same pattern as Phase 1 (int_product_analyst) |
| **Test training time assumption early** | Assumption A-002 said "< 60s". Reality was 31 min. Changed from daily to weekly before shipping |

## Known Limitations

| Limitation | Impact | Resolution |
|-----------|--------|------------|
| Training takes 31 min on XSMALL | Weekly schedule instead of daily | Bump to SMALL for daily (~$5/mo more) |
| Backtest not run | No MAPE accuracy number | Run `test_forecast_backtest.py` manually |
| Some calibers predict beyond 30 days | Chart filtered to 30d; F_FORECAST has extra rows | Clean in stored procedure or accept |
| Revenue model trained manually | Not in the weekly Task yet | Add to SP_TRAIN_FORECAST |
| No forecast accuracy monitoring | No alerts if model degrades | Phase 2b or 3 |
