## Summary

Added Page 5 "Customer Intelligence" to the Streamlit Sales Dashboard — a dedicated view of customer health that was previously inaccessible without writing SQL. The page combines an RFM segment health table (all 17 classifications with customer counts and lifetime value), a color-coded distribution chart, top 10 at-risk customers by LTV, and a CORTEX.COMPLETE executive summary banner powered by `gemini-2-5-flash`. The LLM call is cached 10 minutes (matching the dbt build cadence) and degrades gracefully on failure. Net new cost: ~$0.15/month. No new infra, no new dbt models.

## Lessons Learned

### Process

- **Brainstorm → Define → Design → Build was the right sequence here.** The brainstorm surfaced the MoM delta constraint early (no historical snapshots), preventing scope creep during build. Without Phase 0, that constraint would have been discovered mid-build.
- **5 discovery questions was the right depth** for a single-page Streamlit feature. No clarification was needed after the brainstorm — the DEFINE wrote itself cleanly.

### Technical

- **`gemini-2-5-flash` works in Snowflake via CORTEX.COMPLETE cross-region routing** — no special configuration needed. One-line constant makes it trivial to swap to `llama3.1-70b` if ever needed.
- **SQL-side aggregation + Python prompt assembly is the right pattern** for LLM-on-structured-data tasks. Keeping aggregation and LLM calls as separate cached functions means both can be developed, debugged, and cached independently.
- **`str | None` return type with `try/except Exception` on the LLM call** is the correct graceful degradation pattern for CORTEX.COMPLETE in Streamlit. Never let an LLM failure crash the page.
- **The segment constants (`CONCERNING_SEGMENTS`, `POSITIVE_SEGMENTS`) as Python sets** is cleaner than a SQL CASE or a dbt seed for 17 stable values. Matches how Page 1 handles `DEFAULT_STATUSES`.
- **Single-quote escaping** (`prompt.replace("'", "''")`) is necessary when embedding a Python f-string into a Snowflake SQL string literal. Easy to miss, causes a syntax error at runtime.

### Communication

- **Recommending Approach C (structured dashboard + thin LLM glue)** over A/B upfront saved a cycle. User confirmed it immediately — the reasoning was clear.
- **Surfacing the MoM delta constraint explicitly** (no dbt snapshot = can't do deltas) during brainstorm meant it was consciously deferred, not accidentally forgotten.
- **Model choice conversation** (which CORTEX.COMPLETE model?) was worth having — user preferred `gemini-2-5-flash` over `llama3.1-70b` after seeing the full model table. Documenting this as a swappable constant made the decision low-stakes.

### Tools & Libraries

- **`dark_dataframe()` with `fmt` dict** handles all table formatting needs cleanly — no custom HTML table needed beyond what the utility already provides.
- **`go.Bar(orientation="h")` with `automargin=True`** on the y-axis is the right pattern for long label names in horizontal bar charts (avoids label truncation).
