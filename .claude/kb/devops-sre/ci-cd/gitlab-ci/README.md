## Why GitLab CI?

GitLab CI/CD is an integrated pipeline platform within GitLab, offering a single application for the entire DevOps lifecycle. Its `.gitlab-ci.yml` configuration provides powerful pipeline definitions with stages, artifacts, and environments.

**When to use GitLab CI:**
- Teams already on GitLab for source control
- Need for integrated DevSecOps (SAST, DAST, dependency scanning)
- Complex multi-stage pipelines with artifacts
- Self-hosted CI/CD runners for compliance
