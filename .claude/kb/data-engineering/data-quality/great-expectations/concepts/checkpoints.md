# Checkpoints

> **MCP Validated:** 2026-02-19

## What Is a Checkpoint?

A Checkpoint is the primary means for validating data in production. It runs one or more Validation Definitions and triggers Actions based on the results. Checkpoints are the orchestration layer that connects your validation logic to your pipeline infrastructure.

**Flow:** Checkpoint → runs Validation Definitions → produces Results → triggers Actions

## Creating a Checkpoint

```python
import great_expectations as gx
from great_expectations.checkpoint.actions import (
    UpdateDataDocsAction,
    SlackNotificationAction,
)

context = gx.get_context()

# Assume validation_def already exists
checkpoint = context.checkpoints.add(
    gx.Checkpoint(
        name="orders_checkpoint",
        validation_definitions=[validation_def],
        actions=[
            UpdateDataDocsAction(name="update_docs"),
        ],
        result_format="SUMMARY",
    )
)
```

## Running a Checkpoint

```python
# Run all validation definitions
result = checkpoint.run()

# Check overall success
print(result.success)            # True if ALL validations pass
print(result.describe())         # Human-readable summary

# With DataFrame batch parameters
result = checkpoint.run(batch_parameters={"dataframe": df})
```

## Checkpoint Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `name` | str | Unique checkpoint name |
| `validation_definitions` | list | Validation Definitions to run |
| `actions` | list | Actions to trigger after validation |
| `result_format` | str | `BOOLEAN_ONLY`, `BASIC`, `SUMMARY`, `COMPLETE` |

## Built-in Actions

| Action Class | Purpose |
|-------------|---------|
| `UpdateDataDocsAction` | Rebuild Data Docs HTML site |
| `SlackNotificationAction` | Send Slack message via webhook |
| `EmailAction` | Send email notification |
| `MicrosoftTeamsNotificationAction` | Send Teams message |
| `StoreValidationResultAction` | Persist results to store |
| `StoreMetricsAction` | Extract and store metrics |

### Configuring Actions

```python
from great_expectations.checkpoint.actions import (
    UpdateDataDocsAction,
    SlackNotificationAction,
)

checkpoint = gx.Checkpoint(
    name="prod_checkpoint",
    validation_definitions=[validation_def],
    actions=[
        UpdateDataDocsAction(name="update_docs"),
        SlackNotificationAction(
            name="slack_alert",
            slack_webhook="${SLACK_WEBHOOK_URL}",
            notify_on="failure",  # "all", "failure", "success"
        ),
    ],
)
```

## Managing Checkpoints

```python
# Retrieve
checkpoint = context.checkpoints.get("orders_checkpoint")

# List all
all_checkpoints = context.checkpoints.all()

# Delete
context.checkpoints.delete("orders_checkpoint")
```

## Checkpoint Results

```python
result = checkpoint.run()

# Top-level
result.success                  # bool: overall pass/fail
result.describe()               # str: formatted summary

# Access individual validation results
for vd_result in result.run_results.values():
    print(vd_result.success)
    for exp_result in vd_result.results:
        print(exp_result.expectation_config.type)
        print(exp_result.success)
```
