## Key Concepts

| Concept | Description |
|---------|-------------|
| **Data Contract** | A formal, versioned agreement specifying schema, quality rules, SLAs, and ownership between producer and consumer |
| **Schema** | The structural definition of fields, types, constraints, and relationships |
| **SLA** | Service-level agreements on freshness, availability, and response time |
| **ODCS** | Open Data Contract Standard v3.1.2 — YAML-based spec by Bitol (LF AI & Data), now the default format |
| **datacontract-cli** | CLI tool (v0.11.4) for linting, testing, diffing, and exporting data contracts |
| **Contract Testing** | Automated validation that data conforms to the contract at pipeline boundaries |

## Ecosystem Updates (Feb 2026)

- **ODCS v3.1.2** is now the default format generated by `datacontract init`
- **datacontract-cli v0.11.4**: DuckDB is now optional, Spark decimal precision fixes, BigQuery improvements
- **Old "Data Contract Specification" format** (pre-ODCS) is officially deprecated
- **O'Reilly book "Data Contracts"** (Nov 2025) provides comprehensive industry guidance
- The ecosystem is maturing rapidly with growing enterprise adoption
